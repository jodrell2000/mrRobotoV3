# Version 1.3.0 - OpenChat Migration

**Release Date:** TBD  
**Type:** Minor Release

## Overview

Migrated the bot's chat service from CometChat to OpenChat (tt.fm's official CometChat-compatible chat endpoint). OpenChat provides 100% API compatibility with CometChat, requiring only a base URL change. This migration improves service integration with the tt.fm platform while maintaining all existing chat functionality. Additionally, removed Google Cloud dependencies to streamline infrastructure (Oracle Cloud deployment remains unchanged).

## 🔄 Changed

### Chat Service Migration

- **Created:** New `openchatApi.js` service as drop-in replacement for `cometchatApi.js`
- **API Compatibility:** OpenChat is 100% CometChat-compatible (identical endpoints, authentication, responses)
- **Base URL Change:** Updated from `https://{COMETCHAT_API_KEY}.apiclient-us.cometchat.io` to `https://openchat.prod.tt.fm/`
- **Service Updates:** Updated `messageService.js`, `privateMessageService.js`, `groupMessageService.js` to use openchatApi
- **Configuration:** Added `OPENCHAT_BASE_URL` environment variable (defaults to `https://openchat.prod.tt.fm/`)
- **Testing:** Created comprehensive test suite with 124+ passing tests for OpenChat services

## Migration Guide

### For New Deployments

1. Use the standard setup process from [SETTING_UP_YOUR_ENVIRONMENT.md](../SETTING_UP_YOUR_ENVIRONMENT.md)
2. The `COMETCHAT_AUTH_TOKEN` obtained from the Hangout gateway works with OpenChat
3. Optional: Set `OPENCHAT_BASE_URL` in `.env` (defaults to `https://openchat.prod.tt.fm/`)

### For Existing Deployments

See [docs/migrations/1.3.0_OPENCHAT_MIGRATION_GUIDE.md](../migrations/1.3.0_OPENCHAT_MIGRATION_GUIDE.md) for detailed migration instructions.

**Quick Migration Steps:**

1. **Regenerate Auth Token:**
   ```bash
   # Get new auth token from gateway using your bot token
   # Follow Step 5 in SETTING_UP_YOUR_ENVIRONMENT.md
   ```

2. **Update .env:**
   ```env
   # Add new variable (optional - uses default if omitted)
   OPENCHAT_BASE_URL="https://openchat.prod.tt.fm/"
   
   # Update auth token with newly generated value
   COMETCHAT_AUTH_TOKEN="your-new-auth-token-here"
   ```

3. **Deploy updated code:**
   ```bash
   # Oracle Cloud
   ORACLE_IP=YOUR_VM_IP ./scripts/deploy-to-oracle.sh
   
   # Docker (local)
   docker-compose down
   git pull
   docker-compose up -d
   ```

4. **Verify:**
   - Check logs for successful startup
   - Test message sending/receiving in hangout
   - Verify group operations work

### Rollback Procedure

If issues arise, you can rollback to CometChat:

1. **Uncomment cometchatApi.js:**
   ```bash
   # Remove // comments from all lines in src/services/cometchatApi.js
   sed -i '' 's|^// ||g' src/services/cometchatApi.js
   ```

2. **Update serviceContainer.js:**
   ```javascript
   // Change this:
   const openchatApi = require('./openchatApi.js');
   
   // Back to this:
   const cometchatApi = require('./cometchatApi.js');
   
   // And update the setRetryService call
   cometchatApi.setRetryService(retryService);
   ```

3. **Update service imports:**
   ```bash
   # messageService.js, privateMessageService.js, groupMessageService.js
   # Change: require('./openchatApi.js')
   # Back to: require('./cometchatApi.js')
   ```

4. **Redeploy with original auth token**

## Technical Details

### API Compatibility

OpenChat maintains 100% compatibility with CometChat REST API v3:

**Identical Endpoints:**
- `/v3.0/messages` - Send messages
- `/v3/groups/{id}/members` - Join/leave groups
- Message structure, custom data, authentication headers all unchanged

**Authentication:**
- Same header structure: `authtoken`, `appid`, `apikey`
- COMETCHAT_AUTH_TOKEN works with OpenChat (regeneration recommended but not required)

### Implementation Details

**openchatApi.js Structure:**
- `BASE_URL` - Configurable via OPENCHAT_BASE_URL
- `buildCustomData()` - Constructs message custom data (avatarId, userName, color, mentions, userUuid, badges, id)
- `buildPayload()` - Creates message payload for API
- `sendMessage()` - POST to `/v3.0/messages` with retry logic
- `joinChatGroup()` - POST to `/v3/groups/{roomId}/members`
- `leaveChatGroup()` - DELETE from `/v3/groups/{roomId}/members/{botUid}`
- `fetchMessages()` - GET messages with retry/circuit breaker
- `markConversationAsRead()` - POST mark as read
- `setRetryService()` - Inject retry service dependency

**Retry Operations:**
- `openchat-sendMessage` - Message sending with retry
- `openchat-joinGroup` - Group join with retry
- `openchat-leaveGroup` - Group leave with retry
- `openchat-fetchMessages` - Message fetch with retry/circuit breaker
- `openchat-markAsRead` - Mark read with retry

### Performance

No performance impact expected:
- Same API structure and endpoints
- Existing retry logic and circuit breaker patterns maintained
- Same timeout and error handling behavior

### Backwards Compatibility

**Breaking Changes:** None for end users

**Code Changes Required:** None - this is an infrastructure update

**Configuration Changes:**
- Optional: Add `OPENCHAT_BASE_URL` to `.env` (uses default if omitted)
- Recommended: Regenerate `COMETCHAT_AUTH_TOKEN` via gateway
