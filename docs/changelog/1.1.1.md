# Version 1.1.1 - Timezone Support Fix

**Release Date:** 12 May 2026  
**Type:** Patch Release (Bug Fix)

## Overview

Fixed timezone handling in Docker containers by adding timezone database support to the Alpine Linux base image.

## What's Fixed

### 🐛 Timezone Support

**Problem:**
- The `{currentTime}` token was displaying system time instead of the configured timezone
- Alpine Linux base image lacks timezone data by default
- Configuration was stored correctly but `toLocaleTimeString()` couldn't find timezone databases

**Solution:**
- Added `tzdata` package installation to Dockerfile
- Timezone configuration now works correctly across all supported timezones

**Example:**
```
Configuration: America/Los_Angeles
Before: Shows 09:39 (UK time)
After: Shows 01:39 (LA time)
```

## Technical Changes

### Modified Files
- `Dockerfile`: Added `RUN apk add --no-cache tzdata` to install timezone database

### Impact
- **Breaking Changes:** None
- **New Dependencies:** tzdata (Alpine package)
- **Migration Required:** None - redeploy only

## Upgrade Instructions

1. Pull the latest v1.1.1 image
2. Restart the container

No configuration changes or data migration required.

## Related Configuration

The following tokens now work correctly with configured timezones:
- `{currentTime}` - Current time in configured timezone
- `{currentDate}` - Current date in configured timezone
- `{currentDayOfWeek}` - Current day in configured timezone
- `{greetingTime}` - Time-based greeting (morning/afternoon/evening)

Configure timezone using:
```
!token add timezone "America/Chicago"
!token add locale "en-US"
!token add timeFormat "24"
```
