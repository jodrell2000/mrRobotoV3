## ✨ New Features & Improvements

### Google Cloud Run Deployment

Complete infrastructure for deploying Mr. Roboto V3 to Google Cloud Run with comprehensive documentation.

#### New Files

- **`docs/CLOUD_HOSTING.md`** - Complete guide covering Google Cloud Run, Oracle Cloud, and alternative platforms
- **`scripts/deploy-to-cloudrun.sh`** - Automated deployment script with environment variable parsing

#### Features

- **Automated Deployment Script**
  - Validates prerequisites (gcloud CLI, authentication, .env file)
  - Fetches available GHCR tags for version selection
  - Parses `.env` file automatically (no manual secret configuration)
  - Copies image to Artifact Registry via Cloud Build (no local Docker required)
  - Deploys with optimal settings for WebSocket persistence (`--min-instances 1`, `--no-cpu-throttling`)
  - Shows management commands after deployment

- **`--upload-data` Flag**
  - Upload local data directory to GCS before deployment
  - Ensures deployed bot starts with latest local changes
  - Validates GCS bucket configuration from `.env`
  - Uses `gcloud storage rsync` for efficient sync

#### Documentation

- Step-by-step setup guide from Google Cloud account to deployed bot
- Cost breakdown with free tier analysis (expected: $0/month)
- Comparison with Oracle Cloud, Fly.io, Railway, and Render
- Troubleshooting section for common deployment issues
- Cost monitoring and billing alert setup

#### Usage

```bash
# Deploy with environment variables from .env
bash scripts/deploy-to-cloudrun.sh

# Deploy and upload local data to GCS first
bash scripts/deploy-to-cloudrun.sh --upload-data
```

### Cloud Storage Integration

Persistent data storage using Google Cloud Storage to solve Cloud Run's stateless container limitation.

#### New Files

- **`src/services/cloudStorageService.js`** - Bidirectional sync service for GCS
- **`src/tasks/dailyCloudSyncTask.js`** - Automated daily backup task
- **`scripts/sync-cloud-data.sh`** - Manual sync and download script

#### Features

- **Automatic Data Sync**
  - Downloads data from GCS on bot startup
  - Daily backups to GCS at 3 AM (configurable)
  - Preserves SQLite database, configuration files, and all data
  - Within Google Cloud free tier (5GB storage, 5000 operations/month)

- **CloudStorageService**
  - Methods: `uploadFile()`, `downloadFile()`, `syncToCloud()`, `syncFromCloud()`
  - Graceful degradation when GCS_BUCKET_NAME not configured
  - Comprehensive error handling and logging
  - Integration with serviceContainer

- **Manual Sync Script**
  - Creates timestamped backups of local data before downloading
  - Downloads all files from GCS to local
  - Interactive prompts for bucket name and sync trigger
  - Useful for local development and testing

#### Configuration

Add to your `.env` file:
```bash
GCS_BUCKET_NAME=mrroboto-data-YOUR_PROJECT_ID
```

Create bucket:
```bash
gcloud storage buckets create gs://mrroboto-data-YOUR_PROJECT_ID \
  --location=europe-west1 \
  --uniform-bucket-level-access
```

### Gemma 4 Model Support

Switched from Gemma 3 models to newer Gemma 4 models with enhanced reasoning capabilities.

#### Changes

- **Primary Model:** `gemma-3-27b-it` → `gemma-4-31b-it`
- **Secondary Model:** `gemma-3-12b-it` → `gemma-4-26b-a4b-it`
- **Model Discovery:** Auto-discovery now filters to `gemma-4` prefix only
- **Thinking Config:** Added `thinkingConfig` with `includeThoughts: true` and `thinkingLevel: "HIGH"` for improved reasoning

#### Benefits

- Enhanced reasoning capabilities via Gemma 4's thinking features
- More consistent responses with better context understanding
- Automatic fallback chain ensures availability

### ML Configuration Improvements

Refined Google GenAI API parameters to reduce response repetition and improve quality.

#### Changes

- **Temperature:** `2.0` → `0.8` (reduced from overly random value)
- **Top-P:** Added `0.8` (nucleus sampling for diversity)
- **Top-K:** Added `20` (token sampling limit)
- **Max Output Tokens:** Added `1024` (explicit response length limit)
- **Thinking Config:** Added for Gemma 4 reasoning capabilities

#### API Parameter Investigation

- Confirmed Google GenAI API does not support `presencePenalty` or `frequencyPenalty` (OpenAI-specific)
- Identified supported parameters: `temperature`, `topP`, `topK`, `maxOutputTokens`, `stopSequences`
- Updated configuration to use only supported parameters
- Fixed inline comments to accurately reflect actual values

#### Testing

- All 41 ML service tests passing
- Updated test expectations to match new configuration
- Validates model selection, configuration parameters, and thinking config

## 🚀 Deployment Notes

This release enables production deployment to Google Cloud Run with:

1. **Zero-cost hosting** within Google Cloud free tier
2. **Persistent data** via Cloud Storage integration
3. **24/7 availability** with WebSocket stability
4. **Automated backups** daily at 3 AM
5. **Simple deployment** via included script

### Migration from Local Docker

1. Set up Google Cloud project and billing (see CLOUD_HOSTING.md)
2. Create GCS bucket: `gcloud storage buckets create gs://mrroboto-data-YOUR_PROJECT_ID`
3. Add `GCS_BUCKET_NAME` to your `.env` file
4. Upload local data: `bash scripts/deploy-to-cloudrun.sh --upload-data`
5. Monitor logs: `gcloud beta logging tail "resource.type=cloud_run_revision"`
