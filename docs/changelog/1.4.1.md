# v1.4.1 - Mistral LLM Backend & Chat Pagination Fix

**Release Date**: June 30, 2026

## Overview
This release adds Mistral AI as a multi-backend ML system alongside Gemma, enabling flexible AI provider switching. Additionally, it fixes a critical chat pagination bug that was causing messages to get stuck in a 50-message loop.

## Major Changes

### 🤖 Mistral AI Backend Integration
- **New ML Backend**: Full Mistral AI support as alternative to Google Gemma
- **Backend Switching**: New `!switchllm` command (owner-only) to switch between backends
- **Status Command**: `!llmstatus` shows current active backend and availability
- **Fallback Chain**: Automatic failover to Gemma if Mistral fails (configurable)
- **Persistent Selection**: Active backend choice saved across restarts

**Files Modified**:
- Added [mistralBackend.js](../src/services/mistralBackend.js) - Mistral provider implementation
- Updated [machineLearningService.js](../src/services/machineLearningService.js) - Multi-backend orchestration
- Added [handleSwitchLLMCommand.js](../src/commands/Bot%20Commands/handleSwitchLLMCommand.js) - Backend switching
- Added [handleLLMStatusCommand.js](../src/commands/Bot%20Commands/handleLLMStatusCommand.js) - Status reporting

### 🐛 Critical Chat Pagination Fix
- **Root Cause**: Message tracking only updated when commands were processed, not on every fetch
- **Impact**: Bot got stuck requesting same 50-message batch repeatedly
- **Solution**: Now updates `lastMessageID` and `fromTimestamp` for every message batch, ensuring pagination window advances correctly
- **Result**: Messages now progress normally through the entire chat history

**Files Modified**:
- [bot.js](../src/lib/bot.js) - `_fetchNewMessages()` now updates tracking on every fetch
- [groupMessageService.js](../src/services/groupMessageService.js) - Enhanced timestamp tracking

## Configuration

### New Environment Variables
```bash
MISTRAL_API_KEY=your-mistral-api-key-here  # Optional - for Mistral backend
```

**To obtain a Mistral API Key:**
1. Visit https://v2.auth.mistral.ai/login
2. Click "API Keys" in the left sidebar
3. Click "Add a new key" → give it a name → Create
4. Copy the key immediately (not shown again)
5. Add to `.env` file

### New Commands (Owner-Only)
- `!switchllm gemma` - Switch to Google Gemma backend
- `!switchllm mistral` - Switch to Mistral AI backend
- `!llmstatus` - Check current backend status

## Technical Details

### Message Tracking Enhancement
- Timestamp-based pagination now fully implemented
- Service state and instance state kept synchronized
- All messages (not just commands) trigger tracking updates
- Enables correct API window positioning on next fetch cycle

### Backend System Architecture
- **MachineLearningService**: Orchestrator for multi-backend support
- **GemmaBackend**: Google Gemma model provider
- **MistralBackend**: Mistral AI model provider
- **Fallback Chain**: Configurable priority order (default: Mistral → Gemma)
