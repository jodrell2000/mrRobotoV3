# Version 0.9.8_beta - Session Theme Tokens

**Release Date:** 2026-03-14  
**Type:** Feature Release (Beta)

## 🚀 Overview

This release introduces a pair of owner-settable tokens — `{theme}` and `{readTheme}` — that let you announce and manage a session theme (e.g. "Tonight we're playing all 90s house music") across any bot message template. A new `!edit clear` subcommand is also included, making it easy to unset any editable message.

## ✨ New Features & Improvements

### Session Theme Tokens

#### `{theme}` Token
- **Editable message**: Set with `!edit theme <value>` (requires OWNER role)
- **Persists across restarts**: Stored in `data/botConfig.json` via DataService
- **Cleared cleanly**: Use `!edit clear theme` to remove the theme — the token resolves to an empty string when unset
- **Available anywhere**: Can be used in any message template that supports tokens (welcome message, now playing, etc.)

#### `{readTheme}` Token
- **Smart self-suppression**: Returns an empty string when no theme is set, so surrounding message text doesn't look odd
- **Editable template**: The announcement phrasing is itself configurable — set with `!edit readTheme <template>`, e.g. `!edit readTheme Tonight's theme is {theme}`
- **Default template**: If `readTheme` has never been edited, the default is `The theme is currently {theme}`
- **Persists across restarts**: Template stored in `data/botConfig.json` via DataService
- **Recommended usage**: Use `{readTheme}` in your welcome message rather than hardcoding the announcement text — it will automatically disappear when no theme is active

#### Example Workflow

```
// Set a theme for the night
!edit theme 90s house music

// Welcome message containing {readTheme} now produces:
// "Good evening Username and welcome to Hangout FM. The theme is currently 90s house music"

// At the end of the night, clear the theme
!edit clear theme

// Welcome message now produces (readTheme returns empty string):
// "Good evening Username and welcome to Hangout FM. "

// Customise the readTheme announcement phrasing
!edit readTheme Tonight's theme is {theme}
// Now produces: "Tonight's theme is 90s house music"
```

### `!edit clear` Subcommand

- **New subcommand**: `!edit clear <messageType>` unsets any editable message, restoring it to its default or empty value
- **Usage**: `!edit clear theme`, `!edit clear welcomeMessage`, `!edit clear readTheme`, etc.
- **Requires OWNER role**
- **Updated help text**: `!edit` usage now shows `clear` alongside `list`, `show`, and the update syntax

### Per-User Personalised Welcome Messages

Each DJ can now have their own custom welcome message(s) and image(s) that are served at random when they join the hangout. If no personalised data exists for a joining user, the standard `welcomeMessage` template is used as a fallback.

#### How it works

- Personalised data is stored in `data/welcomeMessages.json`, keyed by user UUID
- Each entry holds a `messages` array and a `pictures` array — the same structure used by chat commands
- When a user joins, one message is picked at random; if pictures are present, one is also picked at random
- The `{username}` token is available inside personalised messages and is replaced with a mention of the arriving user
- When a picture is selected, it is sent as an image message (same as `sendGroupPictureMessage`); when there is no picture, it is sent as a plain text message

#### `!editWelcome` Command

Manages per-user personalised welcome data. Requires **OWNER** role.

| Subcommand | Example | Description |
|---|---|---|
| `list` | `!editWelcome list` | List all users who have custom welcome data, with message and image counts |
| `show` | `!editWelcome show "Happy Gilmore"` | Show all messages and images stored for a user |
| `addMessage` | `!editWelcome addMessage "Happy Gilmore" Hey {username}!` | Add a welcome message for a user |
| `removeMessage` | `!editWelcome removeMessage "Happy Gilmore" Hey {username}!` | Remove an exact-match message |
| `addImage` | `!editWelcome addImage "Happy Gilmore" https://media.giphy.com/img.gif` | Add a welcome image URL |
| `removeImage` | `!editWelcome removeImage "Happy Gilmore" https://media.giphy.com/img.gif` | Remove an exact-match image URL |
| `remove` | `!editWelcome remove "Happy Gilmore"` | Remove all custom welcome data for a user |

#### Nickname handling

- User identifiers can be a **UUID** or a **nickname**
- Nicknames containing spaces must be wrapped in **double quotes**: `"Happy Gilmore"`
- Single-word nicknames work without quotes: `!editWelcome show JohnDoe`
- Lookups are **case-insensitive** and tolerate extra whitespace (e.g. double spaces) in stored names
- If a nickname is not found, the bot performs a **fuzzy match** across live state and the database and returns up to 3 suggestions, each quoted so they can be copy-pasted directly

#### Example workflow

```
// Add a couple of messages and an image for a user
!editWelcome addMessage "Happy Gilmore" Prepare your facial hair {username} — the surgery is open!
!editWelcome addMessage "Happy Gilmore" Welcome back {username}, good to see you!
!editWelcome addImage "Happy Gilmore" https://c.tenor.com/qosLRuL-U1oAAAAC/69.gif

// Check what's stored
!editWelcome show "Happy Gilmore"

// Remove one message
!editWelcome removeMessage "Happy Gilmore" Welcome back {username}, good to see you!

// Remove all data for the user
!editWelcome remove "Happy Gilmore"
```

#### Tokens available in personalised messages

| Token | Replaced with |
|---|---|
| `{username}` | A @mention of the arriving user |

> Other token service tokens (e.g. `{hangoutName}`, `{theme}`) are also resolved if the token service is available.
