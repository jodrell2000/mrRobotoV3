# Version 1.1.3 - Personality Selective Merging

**Release Date:** 14 May 2026  
**Type:** Enhancement  
**Impact:** Improves personality switching flexibility

## Problem Statement

In versions prior to 1.1.3, activating a personality would overwrite ALL values in each section, even when the personality had empty or unset values. This made it impossible to:

- Switch personalities while keeping the current theme
- Maintain custom triggers across personality switches
- Preserve specific settings that weren't part of the personality definition

**Example Scenario (v1.1.2 and earlier):**
```bash
# Current state
!edit theme "Rock Night - Heavy Hitters"
!trigger add contains "awesome" "That's what I'm talking about!"

# Activate a personality with no theme and no triggers
!personality activate "Minimal DJ"  # Has empty theme and empty triggers

# Result:
# - Theme is cleared (becomes "")
# - Triggers are cleared (becomes {})
# - Lost your current settings!
```

This forced users to either:
1. Include all settings in every personality (lots of duplication)
2. Manually restore settings after switching personalities
3. Avoid switching personalities to preserve settings

## Solution

Enhanced personality activation with **selective merging**: only apply values that are explicitly set in the personality, preserving existing values when the personality has empty/unset fields.

**Merging Logic:**
- **String values**: Empty or whitespace-only strings are skipped (existing value preserved)
- **Arrays**: Empty arrays are skipped (existing array preserved)
- **Objects**: Empty objects are skipped (existing object preserved)
- **Explicitly set values**: Any non-empty value overwrites the existing value

**New Behavior (v1.1.3):**
```bash
# Current state
!edit theme "Rock Night - Heavy Hitters"
!trigger add contains "awesome" "That's what I'm talking about!"

# Activate a personality with no theme and no triggers
!personality activate "Minimal DJ"  # Has empty theme and empty triggers

# Result:
# - Theme remains "Rock Night - Heavy Hitters" ✅
# - Triggers remain intact ✅
# - Personality's AI instructions and messages are applied
```

## What Gets Merged

### Instructions
- `MLPersonality` - AI personality description
- `MLInstructions` - AI behavioral instructions
- **Merge behavior**: Empty strings are skipped, existing values preserved

### Editable Messages
- `welcomeMessage`
- `nowPlayingMessage`
- `justPlayedMessage`
- `theme` ⭐ (common use case)
- `readTheme`
- **Merge behavior**: Empty strings are skipped, existing values preserved

### Configuration
- `timezone`, `locale`, `dateFormat`, `timeFormat`, `botName`, etc.
- **Merge behavior**: Object merge (from v1.1.2), missing keys preserve existing values

### ML Questions
- `popfactsQuestion`, `whatyearQuestion`, `meaningQuestion`, etc.
- **Merge behavior**: Object merge, only non-empty question values are updated

### Disabled Commands/Features
- `disabledCommands` array
- `disabledFeatures` array
- **Merge behavior**: Empty arrays are skipped, existing arrays preserved

### Triggers
- Chat trigger definitions (exact, contains, regex)
- **Merge behavior**: Empty objects are skipped, existing triggers preserved

### Custom Tokens
- User-defined token definitions
- **Merge behavior**: Empty objects are skipped, existing tokens preserved

## Use Cases

### Theme-Independent Personalities

Create personalities that work with any theme:

```bash
# Save personalities without themes
!edit theme ""
!personality save "Energetic DJ" "High energy personality"
!personality save "Chill DJ" "Relaxed laid-back personality"

# Set a theme and switch between personalities
!edit theme "80s Night"
!personality activate "Energetic DJ"  # Theme remains "80s Night"
!personality activate "Chill DJ"      # Theme still "80s Night"

# Change theme while keeping personality
!edit theme "Jazz Hour"
# "Chill DJ" personality continues with new theme
```

### Trigger-Independent Personalities

Maintain custom triggers across personality switches:

```bash
# Set up custom triggers once
!trigger add contains "love this" "Glad you're enjoying it!"
!trigger add regex "awesome|amazing" "Thanks! 🎵"

# Switch personalities without losing triggers
!personality activate "Morning DJ"  # Triggers preserved
!personality activate "Evening DJ"  # Triggers still there
```

### Base + Override Pattern

Create a base personality and specialized variants:

```bash
# Base personality with core settings (no theme, no triggers)
!personality save "Base DJ" "Core AI personality and messages"

# Activate base then add specifics
!personality activate "Base DJ"
!edit theme "Rock Night"
!trigger add contains "rock on" "🤘"
!personality save "Rock DJ" "Rock-themed variant"

# Base personality remains theme-agnostic
!personality activate "Base DJ"  # No theme, can add any
```

### Partial Personality Updates

Update specific aspects without resaving everything:

```bash
# Current personality with many settings
!personality activate "Full Setup"

# Make changes to just ML personality
!edit MLPersonality "New personality text..."
!edit theme ""  # Clear theme in personality

# Update personality - theme is removed, everything else preserved
!personality update
```

## Technical Changes

### New Helper Function

Added `shouldApplyValue()` helper function to determine if a value should be applied:

```javascript
function shouldApplyValue(value) {
    if (value === undefined || value === null) return false;
    if (typeof value === 'string' && value.trim() === '') return false;
    if (Array.isArray(value) && value.length === 0) return false;
    if (typeof value === 'object' && !Array.isArray(value) && Object.keys(value).length === 0) return false;
    return true;
}
```

### Modified Sections in `handleActivatePersonality()`

**Before (v1.1.2):**
```javascript
// Always overwrites, even if empty
await dataService.setValue('Instructions.MLPersonality', personality.instructions.MLPersonality);
for (const [key, value] of Object.entries(personality.editableMessages)) {
    await dataService.setValue(`editableMessages.${key}`, value);
}
await dataService.setValue('triggers', personality.triggers);
```

**After (v1.1.3):**
```javascript
// Only apply if value is set
if (shouldApplyValue(personality.instructions.MLPersonality)) {
    await dataService.setValue('Instructions.MLPersonality', personality.instructions.MLPersonality);
}
for (const [key, value] of Object.entries(personality.editableMessages)) {
    if (shouldApplyValue(value)) {
        await dataService.setValue(`editableMessages.${key}`, value);
    }
}
if (shouldApplyValue(personality.triggers)) {
    await dataService.setValue('triggers', personality.triggers);
}
```

### Special Handling for ML Questions

ML questions use object merge to allow partial updates:

```javascript
if (shouldApplyValue(personality.mlQuestions)) {
    const currentMlQuestions = dataService.getValue('mlQuestions') || {};
    const mergedMlQuestions = { ...currentMlQuestions };
    for (const [key, value] of Object.entries(personality.mlQuestions)) {
        if (shouldApplyValue(value)) {
            mergedMlQuestions[key] = value;
        }
    }
    await dataService.setValue('mlQuestions', mergedMlQuestions);
}
```

## Backward Compatibility

This change is **fully backward compatible**:

- Existing personalities continue to work exactly as before
- Personalities with all values set behave identically to v1.1.2
- Only personalities with intentionally empty values benefit from new behavior
- No database migration required
- No changes to personality save/update commands
