# OpenChat Migration Guide (v1.3.0 → v1.3.1+)

**Prerequisites:** This guide assumes you have a working Mr. Roboto V3 bot running a version prior to 1.3.0.

## Overview

Version 1.3.0 migrated from CometChat to OpenChat (tt.fm's official CometChat-compatible endpoint). OpenChat is 100% API-compatible with CometChat - same endpoints, authentication, and responses.

**Version 1.3.1+ Update:** Dynamic token fetching now eliminates the need to manually configure `COMETCHAT_AUTH_TOKEN`.

**Required Changes (v1.3.0):**
1. ~~**Regenerate auth token**~~ - **No longer required in v1.3.1+** (fetched automatically)
2. **Add `OPENCHAT_BASE_URL`** to your `.env` file (still required)

**What's New in v1.3.1+:**
- `COMETCHAT_AUTH_TOKEN` is now fetched automatically from the Gateway API during startup
- Token is configured dynamically using your `BOT_USER_TOKEN`
- Bot will fail fast with clear error messages if token cannot be fetched
- You can remove `COMETCHAT_AUTH_TOKEN` from your `.env` file

**Also Changed:**
- Removed unused Google Cloud dependencies

**Migration Time:** ~5-10 minutes (reduced from v1.3.0)

## Migration Steps

### 1. Backup Current Configuration

```bash
cp .env .env.backup
cp -r data data-backup-$(date +%Y%m%d-%H%M%S)
```

### 2. Update Code

Stop your bot, and update to at least v1.3.0

### 3. Auth Token Configuration

#### For v1.3.1 and later (Automatic - Recommended):

✨ **No action required!** The bot automatically fetches the auth token during startup.

- Remove `COMETCHAT_AUTH_TOKEN` from your `.env` file (it's now optional)
- Ensure `BOT_USER_TOKEN` is valid - it's used to fetch the chat token automatically
- The bot will exit with a clear error if token fetching fails

#### For v1.3.0 (Manual - Legacy):

**Existing auth tokens will NOT work with OpenChat.** You must regenerate.

1. Get a new chat auth token from the gateway using your bot token
2. Follow [Step 5 in SETTING_UP_YOUR_ENVIRONMENT.md](../SETTING_UP_YOUR_ENVIRONMENT.md#step-5-chat-token-configuration-automatic)
3. Update your `.env`:
   ```env
   COMETCHAT_AUTH_TOKEN="your-new-token-here"
   ```

### 4. Add OPENCHAT_BASE_URL (REQUIRED)

Add this to your `.env`:

```env
OPENCHAT_BASE_URL="https://openchat.prod.tt.fm/"
```

### 5. Restart and Verify

Restart your bot and verify:
- Check logs for connection errors
- Test `!ping` in your hangout
- Verify bot responds to commands

### Oracle Cloud Deployment

```bash
ORACLE_IP=YOUR_VM_IP ./scripts/deploy-to-oracle.sh --upload-data
```

## Troubleshooting

### Authentication Errors (401) - v1.3.1+
- **Bot fails to start:** Check that `BOT_USER_TOKEN` is valid and has Gateway API access
- **"Failed to fetch CometChat token" error:** Verify `BOT_USER_TOKEN` is correct in `.env`
- **Network errors:** Ensure connectivity to `gateway.prod.tt.fm`
- Check startup logs for detailed error messages

### Authentication Errors (401) - v1.3.0
- **Most common cause:** Did not regenerate auth token (Step 3 is REQUIRED for v1.3.0)
- Verify using **bot token** (not personal token) when getting auth token
- Check for extra quotes or whitespace in token

### Connection Errors
- Verify `OPENCHAT_BASE_URL="https://openchat.prod.tt.fm/"` is in `.env`
- Ensure network connectivity to `openchat.prod.tt.fm`

### Messages Not Sending
- Verify `HANGOUT_ID` is correct
- Check bot joined the hangout (see logs)
- Test with `!ping` first

## Rollback Procedure

If you need to revert to CometChat:

**Quick Rollback:**
```bash
git checkout v1.2.2  # Or your previous version
cp .env.backup .env
docker-compose down && docker-compose up -d
```

**Manual Rollback (if needed):**

1. Uncomment `cometchatApi.js`:
   ```bash
   sed -i '' 's|^// ||g' src/services/cometchatApi.js
   ```

2. Update `src/services/serviceContainer.js`:
   ```javascript
   const cometchatApi = require('./cometchatApi.js');  // was openchatApi
   cometchatApi.setRetryService(retryService);
   ```

3. Update service imports in `messageService.js`, `privateMessageService.js`, `groupMessageService.js`:
   ```javascript
   const cometchatApi = require('./cometchatApi.js');  // was openchatApi
   ```

4. Restart: `docker-compose down && docker-compose up -d`

## Configuration Reference

**Required Changes to .env:**
```env
# Must regenerate - existing tokens will NOT work
COMETCHAT_AUTH_TOKEN="your-new-token-here"

# Must add - required for OpenChat
OPENCHAT_BASE_URL="https://openchat.prod.tt.fm/"
```

**Unchanged Variables:**
- `COMETCHAT_API_KEY`
- `BOT_UID`
- `HANGOUT_ID`
- `COMMAND_SWITCH`
- All other existing variables
