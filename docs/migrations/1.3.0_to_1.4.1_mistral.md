# Migrating from 1.3.0 to 1.4.1 - Mistral LLM Integration & Chat Fixes

## Overview

Version 1.4.1 introduces support for **Mistral AI as an alternative ML backend**, alongside the existing Gemma support. This allows you to switch between AI providers or use automatic fallback chains. Additionally, this release fixes a critical chat pagination bug that prevented the bot from progressing through messages properly.

**Migration Time**: ~10 minutes

## Key Changes

### New Features
- ✨ **Mistral AI Backend**: Alternative to Google Gemma for ML queries
- 🔄 **Backend Switching**: New `!switchllm` command for owners to switch between backends
- 📊 **Status Command**: `!llmstatus` to check active backend and availability
- 🛡️ **Fallback Chain**: Automatic failover if primary backend fails
- 🐛 **Chat Pagination Fix**: Critical fix for message fetching (bot no longer gets stuck)

### Breaking Changes
- **None** - This is a backward-compatible update

## Required Changes

### 1. Update Your .env File (Optional but Recommended)

Add the Mistral API key to your `.env` file:

```bash
MISTRAL_API_KEY=your-mistral-api-key-here
```

**To obtain a Mistral API Key:**

1. Visit https://v2.auth.mistral.ai/login and sign in or create an account
2. Click **"API Keys"** in the left sidebar
3. Click **"Add a new key"**, give it a name (e.g., "HangBot ML Features")
4. Leave other options as defaults and click **"Create"**
5. Copy the key immediately (it won't be shown again)
6. Paste it into your `.env` file as shown above

> 💡 **Note**: You can leave this blank if you only want to use Gemma, but having both backends provides better reliability.

### 1.5. Add Last.fm API Credentials (Optional but Recommended)

Version 1.4.1 includes music verification features powered by Last.fm. Add these credentials to your `.env` file:

```bash
LASTFM_API_KEY=your-lastfm-api-key-here
LASTFM_SHARED_SECRET=your-lastfm-shared-secret-here
```

**To obtain Last.fm credentials:**

1. Visit https://www.last.fm/api and click **"Get an API account"**
2. Create a free Last.fm account if needed
3. Once logged in, click **"Create an API account"**
4. Fill in your application details (e.g., "HangBot Music Verification")
5. Accept the terms and submit
6. You'll be shown your **API Key** and **Shared Secret**
7. Copy both values into your `.env` file as shown above

> 💡 **Note**: Last.fm API access is completely free with no billing concerns.

### 2. No Configuration Changes Required

Your existing `botConfig.json` and other configuration files continue to work unchanged. The bot defaults to Mistral as the active backend but will fallback to Gemma if not configured.

## Chat Pagination Fix Details

### What Was Fixed
In version 1.3.0 and earlier, the bot would get stuck in a 50-message pagination loop, repeatedly fetching and processing the same batch of messages. This was caused by the pagination window not advancing when batches contained no commands.

### What Changed
- Message tracking now updates on every message fetch, not just when commands are processed
- The `lastMessageID` and `fromTimestamp` are persisted correctly across fetch cycles
- Pagination window advances correctly, allowing the bot to process all incoming messages

### No Action Needed
This fix is automatic and requires no configuration changes.

## Backend Configuration Details

### Default Behavior
- **Active Backend**: Mistral (unless changed)
- **Fallback Chain**: If Mistral fails, automatically tries Gemma
- **Persistence**: Your active backend choice is saved and restored on restart

### Commands for Owners

| Command | Description |
|---------|-------------|
| `!switchllm gemma` | Switch to Google Gemma backend |
| `!switchllm mistral` | Switch to Mistral AI backend |
| `!llmstatus` | Check current backend and availability |

### Troubleshooting

**"Unknown backend" error when switching:**
- Verify you're using `!switchllm gemma` or `!switchllm mistral` exactly
- Only owners can run this command

**Both backends show "unavailable":**
- Ensure you have at least one API key configured (`GOOGLE_AI_API_KEY` or `MISTRAL_API_KEY`)
- Check that your API keys are valid and not expired
- Restart the bot to reinitialize backends

**Chat still shows pagination issues:**
- The fix addresses the root cause, but older cached logs may show old behavior
- Verify the `lastMessageID` is advancing in fresh logs
- Clear any cached message state if applicable
